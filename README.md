# Sorteador de desafios de desenho
Uma pequena ferramenta para me ajudar a desenhar com mais criatividade

## Princípios do Projeto
- Simples antes de inteligente
- Dados separados da lógica
- O sorteio deve inspirar, não limitar
- Melhor uma ideia utilizável do que um contexto perfeito
- Novas categorias devem ser fáceis de adicionar

## Estrutura das Tarefas do Projeto
### 1. Definição das Categorias
- Categorias geralmente independentes
    - Tema: Sobre o que é?
    - Universo: Em que mundo acontece?
    - Local: Onde acontece?
    - Clima: Como o ambiente está?
    - Elemento: O que precisa ter?
    - Personagem: Quem está envolvido?
    - Mistério: Qual é o gancho narrativo?
    - Estilo visual: Como isso deveria parecer?
    - Restrições Criativas: Qual o desafio técnico?
- Categorias que podem complementar outras
    - Profissão: Ele/ela/elu que faz?
    - Emoção: Como se sente?
    - Objetivo: O que quer?
    - Conflito: O que impede?
    - Elemento visual: O que aparece?
    - Detalhe visual: O que chama atenção?
    - Pergunta: O que faz refletir?
### 2. Definição dos Modos de Inspiração
- Definido pela quantidade de categorias escolhidas, mas acho que isso ficaria para quando for algo mais inteligente. 
    - Flash (uma centelha): 1-3 categorias
    - Gear (uma direção): 4-6 categorias
    - Machine (uma situação): 6-10 categorias

### 3. Definição do Sistema de Modelos de Contextos
- Apenas mostrando o resultado do sorteio do que foi escolhido
- Dentro de um pequeno parágrafo, uma combinação por enquanto

### 4. Escolha da Ferramenta
- Inicialmente, com Python

### 5. Estrutura dos dados usados
- Pasta fixa com os dados a serem sorteados

### 6. Criação do MVP simples
- Função principal: o usuário escolhe categorias, sorteia o conteúdo e monta um contexto.
